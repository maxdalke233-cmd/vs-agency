VS AGENCY — 3D IMMERSIVE STORYTELLING WEBSITE BUILD PLAN FOR CLAUDE CODE
=======================================================================

PROJECT OVERVIEW
----------------
Build a complete premium homepage for “VS Agency”, a video/content agency that creates high-performing short-form videos for Instagram accounts, creators, personal brands and local businesses.

The website should feel like a cinematic, immersive, Awwwards-level creative agency homepage. It should not look like a normal template landing page.

Main creative idea:
A realistic 3D cinema camera is the “main character” of the homepage. The camera appears in the hero section and stays present throughout the page. While the user scrolls, the camera moves, rotates, zooms, reveals video frames, transforms the story visually, and guides the visitor through the process of how VS Agency creates content.

The site should use 3D scroll storytelling, GSAP ScrollTrigger, smooth transitions, premium typography, cinematic lighting, dark backgrounds, motion design, floating UI elements, animated video cards and high-end spacing.

The final result should feel like a luxury tech/creative agency website inspired by Apple, Porsche, Awwwards, cinematic production studios, and modern motion design portfolios.

MOTION DESIGN DETAILS
---------------------
General rules:
- Smooth easing only
- Avoid bouncy childish effects
- Use power3.out, power4.out, expo.out
- Text should reveal cleanly, not randomly
- Cards should move subtly, not fly around aggressively
- Use opacity, y, scale and blur carefully

Hero animation timing:
- Logo/navbar fade: 0.4s
- Headline reveal: staggered 0.12s per line
- Subheadline: 0.5s after headline
- Buttons: 0.2s after subheadline
- Camera: scale/rotate in 1.2s

Scroll transitions:
- Camera movement should be scrubbed smoothly with scroll
- Use scrub: 1 or scrub: 1.5
- Pinned story sections can use pin: true only on desktop

Microinteractions:
- Buttons glow on hover
- Cards lift on hover
- Project cards tilt slightly on mouse move
- Navbar background appears after scroll
- Cursor glow optional but keep subtle


RESPONSIVE REQUIREMENTS
-----------------------
Desktop:
- Full 3D storytelling experience
- Pinned sections allowed
- Floating cards and metrics visible
- Large typography

Tablet:
- Reduce camera size
- Keep scroll animations but simplify orbit
- Avoid overcrowding

Mobile:
- Camera still visible but less dominant
- No heavy pinned scroll sections if performance suffers
- Stack all content vertically
- Convert horizontal process to vertical process
- Reduce floating labels/cards count
- Keep page fast and usable

Use CSS media queries and GSAP matchMedia.


PERFORMANCE REQUIREMENTS
------------------------
- Lazy load 3D scene
- Use Suspense fallback
- Use compressed GLB later if model is added
- Avoid huge textures
- Reduce lights and shadows on mobile
- Use devicePixelRatio limit on Canvas, e.g. dpr={[1, 1.5]}
- Avoid too many particles
- Avoid massive DOM animation loops
- Prefer transform and opacity animations
- Respect prefers-reduced-motion

If user has prefers-reduced-motion:
- disable scroll-scrub camera movement
- show static camera
- keep simple fades only

--------------------------
